# 每日国内外股市资讯看板

每天北京时间 07:30 从公开网页和 RSS 筛选 A 股、港股、美股、基金/ETF及相关全球宏观资讯，更新 GitHub Pages，生成多巴胺配色的飞书原生完整看板，并向群里推送 1 张精简卡片。A股、港股、美股、股票、基金和 ETF 六个板块各展示最多 10 条，一条资讯可以进入多个相关板块。全程不调用付费 AI API。

## 本地预览

无需密钥即可运行测试或生成固定示例：

```bash
python3 generate.py --self-test
python3 generate.py --demo
python3 -m http.server 8000 -d docs
```

浏览器打开 `http://localhost:8000`。

## GitHub 配置

1. 新建公开仓库并推送本项目。
2. 在仓库 `Settings → Pages` 中将 Source 设为 **GitHub Actions**。
3. 在 `Settings → Secrets and variables → Actions` 添加：

| 名称 | 必填 | 内容 |
| --- | --- | --- |
| `FEISHU_WEBHOOK_URL` | 推荐 | 飞书测试群自定义机器人 Webhook |
| `FEISHU_WEBHOOK_SECRET` | 否 | 机器人启用签名校验时填写 |
| `FEISHU_APP_ID` | 推荐 | 用于每天创建飞书原生完整看板 |
| `FEISHU_APP_SECRET` | 推荐 | 飞书应用密钥，只保存在 GitHub Secrets |
| `WATCHLIST_JSON` | 否 | 私密自选清单，只用于排序和飞书提示 |

`WATCHLIST_JSON` 示例：

```json
["510300", "0700.HK", {"symbol": "AAPL", "name": "Apple"}]
```

飞书自建应用需开通“创建及编辑新版文档”和“修改云文档权限设置”，并发布一个应用版本。配置完成后，在 `Actions → 北京时间晨报更新 → Run workflow` 手动试运行一次。新工作流使用标准 UTC `23:20` 定时，即每天北京时间 07:20 开始，预留约 10 分钟完成采集、发布和飞书推送，目标在 07:30 前后送达。

飞书原生看板采用互联网链接可编辑权限：获得链接的任何人都可查看、评论、复制、编辑并继续分享。日报每天新建，公开链接不应承载自选清单或其他私密信息。

## 资讯来源与提炼

- 官方来源：中国证监会、上交所、深交所、香港交易所、美联储、美国证监会。
- 财经媒体：东方财富、天天基金、CNBC、MarketWatch 的公开栏目或 RSS，仅保留与股票、基金、ETF直接相关的条目。
- 英文资讯保留原题，中文摘要、关注原因、市场影响与资产标签由确定性规则生成。
- 优先使用最近 36 小时资讯，不足板块最多回溯 7 天，不使用无关内容凑数。
- `docs/mobile.html` 是每天生成的单文件手机备用页，不再二次加载 JSON。

## 隐私与失败策略

- 自选清单和所有密钥只从 GitHub Secrets 读取，不写入公开文件。
- 公开数据会在落盘前移除自选匹配字段。
- 生成失败时保留上一期看板，并尝试向飞书发送失败提示。
- 飞书推送失败不会撤销已经发布的看板。
- 页面及摘要是自动规则汇总，不构成投资建议。
